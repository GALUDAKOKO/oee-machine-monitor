1. ระบบนี้คืออะไร
และทำไมต้องใช้
ปัญหาของโรงงานกลึงทั่วไป:
ไม่รู้ว่าเครื่องจักรทำงานจริงกี่ชั่วโมงต่อวัน · ไม่รู้เวลาที่หายไปกับการ Setup · ไม่รู้ว่าของเสียมาจากไหน · ไม่มีข้อมูลสำหรับวางแผนรับงาน

ระบบนี้ช่วยให้โรงงานเริ่ม เก็บข้อมูลเวลาจริง โดยไม่ต้องซื้ออุปกรณ์แพง ใช้แค่ iPad / มือถือ เปิด browser แล้วกดปุ่มตามขั้นตอน

ข้อมูลที่จะได้:
✅ เวลา Setup จริงต่อ Job
✅ เวลาที่เครื่องทำงานจริง
✅ เวลาที่เครื่องหยุด และสาเหตุ
✅ % ของดีต่อ Job
✅ Availability ของเครื่องแต่ละตัว

เมื่อมีข้อมูลแล้ว จะเห็นว่า เวลาหายไปที่ไหน และจะแก้ได้ตรงจุด

2. OEE คืออะไร
Overall Equipment Effectiveness

OEE วัดว่า "เครื่องจักรใช้งานได้ดีแค่ไหน" เทียบกับที่ควรจะเป็น คิดเป็น % จาก 3 ตัวคูณกัน:

สูตร OEE: OEE = Availability × Performance × Quality (ทั้ง 3 ตัว หน่วยเป็น %) ตัวอย่าง: Availability = 80% (เครื่องทำงาน 80% ของเวลา) Performance = 90% (เร็ว 90% ของ standard) Quality = 95% (ของดี 95%) OEE = 0.80 × 0.90 × 0.95 = 68.4%
World Class OEE
85%+
มาตรฐานโลก
ทั่วไป
40-60%
โรงงานทั่วไป
ต้องปรับปรุง
<40%
มีปัญหาใหญ่

หมายเหตุ: OEE 40-60% ไม่ได้แปลว่าแย่ — แปลว่า ยังมีโอกาสปรับปรุงได้มาก และการรู้ตัวเลขนี้คือก้าวแรกที่สำคัญที่สุด

ตัวชี้วัดวัดอะไรLoss ที่ตรวจจับ
Availabilityเครื่องทำงานได้กี่ % ของเวลาแผนBreakdown, Setup, รอ material
Performanceเร็วแค่ไหนเทียบ standardเครื่องช้า, หยุดสั้นๆ บ่อย
Qualityของดีกี่ % ของทั้งหมดของเสีย, Rework

3. Work Sampling
วิธีจับเวลาแบบไม่รบกวนพนักงาน
หลักการ: แทนที่จะยืนจับเวลาติดๆ ซึ่งพนักงานจะรู้สึกถูกจับตา — ให้เดินผ่านทุก 15 นาที ชำเลืองดู 1 วินาที แล้วกดปุ่มตามที่เห็น

Flow การใช้งาน — ทำตามนี้ทุกรอบ:

1
ตั้งนาฬิกาปลุกทุก 15 นาที
ใช้นาฬิกาโทรศัพท์ตั้ง repeat alarm ทุก 15 นาที ตลอดวันทำงาน
2
เมื่อปลุก → เดินไปที่เครื่อง
ไม่ต้องรีบ เดินปกติ ไม่ต้องแจ้งพนักงาน
3
มองเครื่อง 1-2 วินาที → กดปุ่ม
กด ปุ่มเดียวเท่านั้น ที่ตรงกับสิ่งที่เห็น ณ ขณะนั้น
4
เดินออก ทำงานต่อ
ไม่ต้องคุย ไม่ต้องอธิบาย ทำซ้ำรอบถัดไปเมื่อปลุก
5
กดผิด → กด "↩ Undo" ทันที
แก้ได้เฉพาะการกดล่าสุด (1 ครั้ง) ถ้าเลยไปแล้วให้ทำต่อตามปกติ ค่าเฉลี่ยจะยังใกล้เคียง

สถานะ 4 แบบที่กดได้:

▶ RUN
เครื่องกำลังตัด กลึง หรือกัดอยู่จริง
มีเสียง + เศษโลหะ + แกนหมุน
🔧 SETUP
กำลังตั้งมีด ตั้งศูนย์ จับชิ้นงานใหม่
มีคนอยู่ที่เครื่อง แต่ยังไม่ตัด
⬤ IDLE
เครื่องว่าง ไม่มีคนทำงาน
รองาน รอ material รอพนักงาน
⛔ DOWN
เครื่องเสีย หยุดผิดแผน
ซ่อม รอช่าง มีป้าย "เสีย"
⚠️ กฎสำคัญ:
• กด แค่ 1 ครั้ง ต่อรอบ 15 นาที ต่อเครื่อง
• ถ้าไม่แน่ใจ → กด IDLE ดีกว่าเดา
• ต้องไปเห็นเครื่องจริงก่อนค่อยกด อย่ากดจากที่นั่ง
• ทำต่อเนื่อง ≥ 30 observations = ผลน่าเชื่อถือ (ประมาณ 7.5 ชั่วโมง)

4. วิธีใช้ Form บันทึก
Type A / B / C
เลือก Type ตามลักษณะงาน:
Type A = งานต่างกันทุก Job (ไม่ซ้ำ)
Type B = งานเดิมกลับมาซ้ำ (มี Part No เดิม)
Type C = ทำทีละ 1 ชิ้น ชิ้นงานซับซ้อน ใช้เวลานาน

Flow การกดปุ่ม (เหมือนกันทุก Type):

1
🔧 กด "เริ่ม Setup"
กดทันทีที่พนักงานเริ่มตั้งมีด ตั้งศูนย์ หรือจับชิ้นงานเข้าเครื่อง
ระบบบันทึกเวลา T0
2
▶ กด "เริ่ม Run"
กดทันทีที่เครื่องเริ่มตัดชิ้นงาน (มีเสียงเครื่อง + เศษโลหะ)
ระบบบันทึกเวลา T1 และคำนวณ Setup Time = T1 - T0
3
⏸ กด "หยุด" (ถ้ามีหยุดกลางทาง)
กดถ้าเครื่องหยุดระหว่าง Run เช่น เปลี่ยนมีด รอ material เครื่องขัดข้อง
ระบบจะถาม "สาเหตุที่หยุด" — เลือกให้ตรง
4
⏯ กด "กลับมา Run" (หลังหยุด)
กดทันทีที่เครื่องเริ่มตัดใหม่ ระบบคำนวณ Downtime อัตโนมัติ
กด "หยุด" และ "กลับมา" ซ้ำได้ถ้ามีหยุดหลายครั้ง
5
✅ กด "เสร็จแล้ว (Done)"
กดทันทีที่พนักงานเอาชิ้นงานออกจากเครื่อง
ระบบบันทึก T3 และคำนวณ OEE ให้ทันที
6
กรอกจำนวน + กด "บันทึก"
กรอกจำนวนชิ้นที่ผลิตได้และชิ้นที่เสีย → กด "บันทึก"
ข้อมูลจะถูกเก็บไว้ใน browser
7
🔄 กด "เริ่ม Job ใหม่" (ถ้ามี Job ต่อไป)
Form จะ reset กลับเป็น IDLE แต่เก็บชื่อผู้สังเกตและเครื่องจักรไว้ ไม่ต้องกรอกใหม่
⚠️ สิ่งที่ห้ามทำ:
❌ อย่ากด "เริ่ม Run" ก่อน "เริ่ม Setup" (ระบบจะ lock ปุ่มให้แล้ว)
❌ อย่าลืมกด "Done" — ถ้าลืมข้อมูลจะไม่สมบูรณ์
❌ อย่ากด "หยุด" แล้วไม่กด "กลับมา" — Downtime จะนับต่อไปเรื่อยๆ

5. ตีความผลลัพธ์ OEE
ตัวเลขหมายความว่าอะไร
ตัวชี้วัดดีพอใช้ต้องแก้
Availability>70%50-70%<50%
Performance>85%65-85%<65%
Quality>95%85-95%<85%
OEE รวม>65%40-65%<40%
Setup Ratio<15%15-25%>25%

การแปลผล Setup Ratio:

Setup Ratio = เวลา Setup ÷ เวลาทั้งหมดของ Job

ถ้า Setup Ratio >25% = ต้องลด Setup Time
วิธีแก้: เตรียม tool ล่วงหน้า, ทำ SMED, จัดลำดับงานให้ใช้มีดเดิมได้ต่อเนื่อง

สาเหตุที่ Availability ต่ำที่พบบ่อย:

  • ⏳ รอ material → แก้: ปรับระบบเบิกของล่วงหน้า
  • 🔧 Setup นาน → แก้: เตรียมมีด จัดลำดับงาน
  • ⚙️ เครื่องเสีย → แก้: วางแผน PM ก่อนเสีย
  • 👤 รอพนักงาน → แก้: ปรับ workflow การรับงาน

6. Performance วัดเมื่อไหร่ได้?
ต้องมี Standard Cycle Time ก่อน
Performance = (Standard CT × จำนวนชิ้น) ÷ Run Time × 100%

Standard CT = เวลาที่ควรใช้ต่อชิ้น (นาที/ชิ้น)
ถ้าไม่มี Standard CT → คำนวณ Performance ไม่ได้

ลำดับการวัด (แนะนำ):

1
สัปดาห์ที่ 1-2: วัด Type A ก่อน ไม่ต้องใส่ Est CT
วัด Availability + Quality เท่านั้น ไม่มี Performance ก็ไม่เป็นไร ได้ baseline ของเวลา Run จริง
2
Export CSV → ดูตาราง Avg CT ใน index
ระบบคำนวณ avg Run Time ต่อชิ้นจาก history ให้อัตโนมัติ → นั่นคือ Standard CT เริ่มต้น
3
สัปดาห์ที่ 3+: ใส่ Standard CT ใน Type B
เมื่อ Part No เดิมกลับมา → ใช้ค่า Avg CT จาก step 2 → Performance คำนวณได้แล้ว
4
ปรับ Standard CT ทุก 1 เดือน
เมื่อมีข้อมูล ≥ 5 batch ของ Part เดิม → ใช้ค่าเฉลี่ยที่แม่นขึ้น → Performance แม่นตาม

7. คำถามที่พบบ่อย
FAQ
Q: ลืมกด "เริ่ม Run" ทันที กด late ไป 5 นาที ทำยังไง?
A: กดไปตามเวลาจริงที่กด ข้อมูลจะ off ไปนิดหน่อย แต่บันทึก remarks ไว้ว่า "gd late 5 min" เพื่อให้รู้เมื่อดูย้อนหลัง ไม่จำเป็นต้อง perfect ทุก Job
Q: พนักงานย้ายเครื่องกลางงาน บันทึกยังไง?
A: กด "หยุด" ที่เครื่องเดิม เลือกสาเหตุ "อื่นๆ" แล้ว remarks ว่า "ย้ายเครื่อง" จากนั้นบันทึก Job นี้ว่าเสร็จที่เครื่องเดิม (Done) แล้วเปิด form ใหม่สำหรับเครื่องที่ย้ายไป
Q: 1 Job ผลิตได้หลายชิ้น กรอกอย่างไร?
A: กรอก Total Qty ที่ผลิตได้ทั้ง batch ใน "จำนวนผลิต" และ Total Reject ใน "จำนวนเสีย" ระบบคำนวณ Quality Rate อัตโนมัติ
Q: ข้อมูลจะหายไหมถ้าปิด browser?
A: ข้อมูลที่กด "บันทึก" แล้วจะถูกเก็บใน browser memory (localStorage) ข้อมูลจะอยู่ตราบที่ไม่ได้ล้าง browser data ให้ Export CSV เป็นประจำทุกวันเพื่อสำรองข้อมูล
Q: ใช้กี่คนวัดพร้อมกัน?
A: 1 คนต่อ 2 เครื่องได้สบาย — ใช้ Work Sampling เดินสลับดูทุก 15 นาที และกด Form บันทึก OEE เมื่อ Job เสร็จ
Q: Availability 45% แปลว่าอะไร?
A: แปลว่าจาก 8 ชั่วโมงทำงาน เครื่องทำงานจริงแค่ 3.6 ชั่วโมง ที่เหลือ 4.4 ชั่วโมงหายไปกับ Setup + Downtime + Idle ซึ่งเป็นโอกาสให้ปรับปรุงได้มาก
Q: ต้องวัดนานแค่ไหนถึงจะมีข้อมูลพอ?
A: Work Sampling ควรทำ 3 วันต่อเนื่อง (≥90 observations) จึงจะเชื่อถือได้ สำหรับ Form บันทึก OEE ควรมีอย่างน้อย 10 Jobs ต่อเครื่องก่อนจะ present ผล
